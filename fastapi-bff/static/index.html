Diagrama de Procesos (Secuencia Transaccional)

sequenceDiagram
    autonumber
    actor Cliente as Cliente (Usuario)
    participant Web as Web UI (SPA)
    participant BFF as FastAPI BFF (Python)
    participant Java as Java Backend (Core)
    participant DB as H2 Database (SQL)

    Cliente->>Web: Clic en "Procesar Transacción"
    Note over Web: Valida campos en Frontend (monto, descripción)
    Web->>BFF: POST /api/v1/bff/transactions/transfer (JSON)
    Note over BFF: Valida Token de sesión y formatea payload
    BFF->>Java: POST /api/v1/transactions/transfer (HTTP REST)
    
    activate Java
    Note over Java: Inicia transacción ACID (@Transactional)
    Java->>DB: SELECT * FROM CARD WHERE ID = origen FOR UPDATE
    Note over DB: Bloqueo Pesimista (Evita condiciones de carrera)
    DB-->>Java: Retorna datos de Tarjeta Origen (Bloqueada)
    
    Java->>DB: SELECT * FROM CARD WHERE ID = destino FOR UPDATE
    DB-->>Java: Retorna datos de Tarjeta Destino
    
    Note over Java: Verifica Fondos, Expiración y Estado Activo
    
    alt Datos Válidos & Fondos Suficientes
        Java->>DB: UPDATE CARD SET balance = balance - monto WHERE ID = origen
        Java->>DB: UPDATE CARD SET balance = balance + monto WHERE ID = destino
        Java->>DB: INSERT INTO TRANSACTION (tipo, monto, estado) VALUES (TRANSFER, APPROVED)
        Note over Java: Transacción exitosa, se liberan bloqueos (Commit)
        Java-->>BFF: 200 OK (Transaction Approved)
    else Fondos Insuficientes o Tarjeta Bloqueada
        Java->>DB: INSERT INTO TRANSACTION (tipo, monto, estado) VALUES (TRANSFER, DECLINED)
        Note over Java: Transacción rechazada, se liberan bloqueos (Rollback)
        Java-->>BFF: 400 Bad Request (Error explicativo)
    end
    deactivate Java
    
    BFF-->>Web: Retorna respuesta estructurada y estadísticas actualizadas
    Note over Web: Animación de éxito/error y actualización de UI
    Web-->>Cliente: Muestra comprobante y saldo debitado

Patrones de Calidad, Resiliencia y TI

Backend-For-Frontend (BFF)

Capa intermedia en FastAPI que desacopla el cliente de los microservicios internos. Optimiza payloads, realiza enrutamiento inteligente y gestiona caché/sesiones.

Implementado

Bloqueo Pesimista (ACID)

Implementación de FOR UPDATE en base de datos para transacciones críticas de débito/crédito, garantizando consistencia estricta ante operaciones concurrentes.

Implementado

Enmascaramiento de Datos (PII)

Protección de datos sensibles (números de tarjeta expuestos como **** **** **** 1234 y ocultamiento de CVV) para cumplir con estándares PCI-DSS.

Implementado

Resiliencia y Circuit Breaker

Políticas de retry y fallback del BFF FastAPI que manejan caídas o lentitud en el core Java de forma elegante, manteniendo la UX sin errores de servidor genéricos.

Implementado
